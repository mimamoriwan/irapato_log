# イラパトログ — CLAUDE.md

## プロジェクト概要

**アプリ名**：イラパトログ
**目的**：日常のイライラを記録し、繰り返し登場するものをAIで束ねて「アプリ化する価値があるイライラ」を可視化するツール
**現フェーズ**：MVP（自分専用ツールとして動作確認）

### 解決する課題

```
【従来の問題】
1回イライラする → すぐ「これアプリにできないか」と検討開始（単発でも深掘りしてしまう）

【このアプリ導入後】
1回イライラする → 記録するだけ
2回目       → 記録するだけ
3回目       → 「同じやつだ」とAIが可視化
              → 初めてアイデア検討を開始
```

## 画面構成

### 2タブ構成

```
[ 記録タブ ]  [ 分析タブ ]
```

### 記録タブ

- 開いた瞬間に入力できる（余計な画面をはさまない）
- **音声入力ボタン**：Web Speech APIで音声→テキスト変換
- **テキスト入力**：一言で端的に記入（必須）
- **詳細入力**：状況・背景を追記（任意・折りたたみ式）
- 記録時にAIは使わない。保存するだけ
- 記録データ構造：

```typescript
type IraLog = {
  id: string;          // UUID
  text: string;        // 一言メモ（必須）
  detail?: string;     // 詳細（任意）
  createdAt: string;   // ISO8601
};
```

### 分析タブ

- 「分析する」ボタンを手動で押したときだけAIが動く
- 全件ログをGemini APIに送信してグルーピング
- **3回以上 = アイデアの種として検討する価値あり**
- 非同期実行 + ローディング表示（ログ増加でレスポンスが遅くなるため）
- 出力イメージ：

```
🌱 起動・レスポンスの遅さ　4件　★★★★☆
　→「アプリの起動が遅い」
　→「開こうとしたら時間がかかった」
　→「起動に手間取って忘れた」
　→「メモしようとしたら手間取った」

🌱 情報が見つからない　2件　★★☆☆☆
　→「検索しても欲しい情報が出てこない」
　→「また同じ情報を検索してしまった」
```

## Gemini APIのグルーピング指示（プロンプト設計）

```
以下はユーザーが記録した日常のイライラのログです。
表現が異なっていても、意味・文脈が近いものを同じグループにまとめてください。

出力はJSON形式で、以下の構造にしてください：
{
  "groups": [
    {
      "label": "グループ名（10字以内）",
      "count": 件数,
      "logs": ["ログ1", "ログ2", ...]
    }
  ]
}

グループは件数の多い順に並べてください。
JSONのみ出力し、前置きや説明は不要です。

--- ログ一覧 ---
{{logs}}
```

## 開発ルール

### 設計哲学（必ず守ること）

- **記録の摩擦をゼロに保つ**：記録タブは開いた瞬間に入力できる状態にする
- **記録時にAIを使わない**：保存するだけ。AIは分析タブの手動トリガー時のみ
- **罰を与えない**：記録が途切れても何も言わない
- **シンプルを保つ**：MVPに不要な機能は追加しない

### MVPに含めないもの（追加禁止）

- リアルタイムAI仕分け
- 認証・ログイン
- グループ化のUI操作
- 週次自動分析
- アプリアイデア自動生成機能
- プッシュ通知

### コーディング規約

- TypeScript strict mode を使用
- コンポーネントは関数コンポーネント + Hooks
- localStorage操作は `src/storage.ts` に集約
- Gemini API呼び出しは `src/gemini.ts` に集約
- エラーハンドリングは console.error で十分（MVP段階）

## 実装の優先順位

```
Step 1：記録タブ（テキスト入力 + localStorage保存 + 一覧表示）
Step 2：音声入力ボタン（Web Speech API）
Step 3：詳細入力（折りたたみ式）
Step 4：分析タブ（Gemini API連携 + グルーピング表示）
Step 5：ローディング表示 + エラー表示
```

Step 1が動けばすぐ自分で使い始められる。Step 4まで完成でMVP完成。
